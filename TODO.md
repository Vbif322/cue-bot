## Ближайший месяц — фоновой планировщик джобов

Следующий приоритет сразу после M2 по [ROADMAP.md](ROADMAP.md) (раздел «Доработки турнирного
ядра»). Задачи упорядочены по приоритету: сначала дешёвые независимые quick win'ы, затем
фундамент (сам планировщик), затем его самый ценный потребитель, затем остальное.

- **P0 — quick wins**, без зависимостей от планировщика, можно делать в любой момент:
  - колонка `endDate` в `tournaments` (сейчас только `startDate`) + миграция, поле в визарде
    создания и админ-SPA, отображение диапазона дат;
  - рендер `scheduledAt` в игровом вебе — данные уже есть в типах (`app/src/lib/types.ts`), но
    не выводятся в `Bracket`/`MatchModal` (`app/`).
- **P1 — базовый сервис планировщика**, по образцу `dialogSessionSweep` (`src/index.ts`):
  фундамент, без которого не завести ни один из следующих пунктов.
- **P1 — авто-подтверждение результата по таймауту**: первый и самый ценный потребитель
  планировщика — если оппонент не реагирует на заявленный результат за отведённое время,
  автоматически подтверждать счёт заявителя и уведомлять. В ROADMAP.md прямо помечено как
  кандидат вытащить уже в M2, «если матчи подвисают на реальных турнирах». Логика была в
  удалённом `timeoutService`; строить поверх confirm-флоу M2-7.
  Сделать до или вместе с этим пунктом — **P2 «редактирование результата»** из блока M2-7
  техдолга выше (`matchService.ts`): сейчас при dispute репортёр вводит все фреймы заново;
  как только подтверждение начнёт происходить автоматически, а не по явному клику оппонента,
  споры постфактум станут чаще, а без нормальной правки результата единственный выход из
  спора — полный re-ввод.
- **P2 — остальные потребители планировщика**, следующим спринтом:
  - напоминания о матче за N часов до `matches.scheduledAt` обоим игрокам (тип
    `match_reminder` уже есть в `src/db/schema/notifications.ts`, сейчас шлётся один раз при
    назначении времени через `notifyMatchScheduled`);
  - дедлайны матчей — по истечении срока на сыгранную встречу техпоражение через
    `setTechnicalResult` (`src/services/matchService.ts`) и уведомление.
